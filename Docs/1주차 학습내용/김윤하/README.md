## 1. 개발 기술 학습
### 1. FastAPI 프레임워크 학습
- [[FastAPI/Docs]](https://fastapi.tiangolo.com/) FastAPI 공식문서 자습서를 활용해 기본 개념부터 실제 활용까지 체계적으로 학습했습니다. 
- FastAPI의 주요 특징인 비동기 지원, 타입 힌트 기반의 API 설계, 자동화된 문서화(OpenAPI) 등을 이해했습니다.

1. **프로젝트 구조, 작동원리**
- FastAPI의 기본적인 프로젝트 디렉토리 구조를 이해하고, 모듈화된 설계를 통해 애플리케이션을 개발할 수 있는 방식을 학습했습니다.
    - 라우팅 (Routing): @app.get, @app.post 등과 같은 데코레이터를 사용해 경로를 정의하고, 경로 함수가 어떻게 호출되는지 이해했습니다.
    - 의존성 주입 시스템: FastAPI가 제공하는 Dependency Injection 메커니즘을 통해 외부 리소스나 공통 로직을 경로 함수에 쉽게 주입할 수 있다는 점을 익혔습니다.
    - Swagger UI 및 ReDoc 문서화: FastAPI가 자동으로 API 문서를 생성하며, 이를 활용하여 API 테스트 및 공유를 효율적으로 진행할 수 있음을 학습했습니다.
        ![Swagger](image.png)

2. **RESTful API 설계(CRUD)**
- RESTful API 설계를 학습하며 기본 CRUD(Create, Read, Update, Delete) 기능을 구현했습니다.
    - Create: POST 요청을 통해 데이터 생성.
    - Read: GET 요청으로 단일 데이터 조회와 리스트 조회 구현.
    - Update: PUT 및 PATCH 요청으로 데이터 수정.
    - Delete: DELETE 요청을 통해 데이터 삭제.
    - 요청 및 응답에서 FastAPI의 Pydantic 모델을 활용해 데이터 검증(validation)과 serialization을 처리했습니다.

3. **데이터베이스 연결 및 ORM 사용**
- FastAPI는 Django와 달리 독자적인 ORM이 없으며, SQLAlchemy와 Tortoise ORM 등의 라이브러리를 활용해 데이터베이스 작업을 수행할 수 있음을 학습했습니다.
    - SQLAlchemy를 활용한 DB 연결:
        - SQLAlchemy 엔진을 사용해 MariaDB를 연결하는 실습을 진행했습니다.
        - WebRTC 기술을 사용할 계획이기 때문에 높은 성능과 확장성을 제공하는 MariaDB를 선택했습니다.
        - DB 세션 관리, 테이블 생성 및 기본적인 쿼리 작성법을 익혔습니다.
    - Pydantic과의 통합: ORM 모델과 Pydantic 모델 간의 데이터 변환 방법을 학습했습니다.
    - Alembic: 데이터베이스 마이그레이션 도구인 Alembic의 설정 및 사용 방법을 이해했습니다.

4. **JWT 인증 기능 학습**
- FastAPI에서 JWT (JSON Web Token) 인증의 개념과 작동 원리를 학습했습니다.
    - JWT의 개념:
        - JWT는 헤더, 페이로드, 서명으로 구성된 문자열로, 사용자 식별 정보 및 권한 정보를 담아 안전하게 전달할 수 있는 인증 방식입니다.
        - Base64Url로 인코딩된 JSON 데이터를 사용하며, 서버가 상태 정보를 유지하지 않아도 된다는 점에서 RESTful API와 잘 어울립니다.
    - JWT의 작동 원리:
        - 클라이언트가 로그인 정보를 서버에 전달하면, 서버는 해당 정보를 기반으로 JWT를 생성하여 클라이언트에 반환.
        - 이후 클라이언트는 API 호출 시 헤더에 JWT를 포함하여 인증을 진행.
        - 서버는 받은 JWT를 검증하여 유효성을 확인한 뒤 요청을 처리.
    - FastAPI에서 JWT 사용 흐름:
        - PyJWT 라이브러리: FastAPI에서 JWT 생성과 검증을 지원하는 라이브러리로, 이를 사용해 토큰 처리 방식을 이해했습니다.
        - Pydantic 모델과 연계: 사용자 인증 요청 시, Pydantic 모델을 활용해 입력 데이터 검증 및 오류 처리 방식을 학습했습니다.
    - JWT 구성 요소:   
        - 헤더(Header): 토큰의 타입과 서명 알고리즘 정보를 포함.   
        -페이로드(Payload): 사용자 식별자, 권한 정보, 만료 시간 등 클라이언트와 서버 간에 필요한 데이터를 포함.
        - 서명(Signature): 토큰 위변조를 방지하기 위한 해시값으로, 비밀 키를 기반으로 생성.


### 2. WebRTC 학습
- WebRTC 개발을 위해 기본 개념과 주요 기술에 대해 학습하며, 실시간 커뮤니케이션을 구현하기 위한 핵심 요소를 이해했습니다.

1. 시그널링, WebSocket
    - 시그널링(Signaling)의 역할:
        - WebRTC에서 P2P 연결을 설정하기 위해 필수적인 과정으로, 연결 초기화, 메타데이터 교환, 연결 상태 관리 등을 담당합니다.
        - WebRTC 자체에는 시그널링 프로토콜이 포함되어 있지 않기 때문에, WebSocket을 사용하여 시그널링을 구현하는 방법을 학습했습니다.
    - WebSocket:
        - WebSocket은 양방향 통신을 지원하며, 클라이언트와 서버 간 실시간 데이터를 교환하는 데 사용됩니다.
        - WebRTC의 시그널링 과정에서 SDP(Session Description Protocol)와 ICE(Interactive Connectivity Establishment) 후보를 주고받는 데 WebSocket을 사용하는 방식을 이해했습니다.
        
2. 네트워크 관련 CS 지식
- WebRTC에서의 네트워크 이해:
    - WebRTC의 핵심은 P2P 연결을 활용하는 것이지만, NAT(Network Address Translation) 및 방화벽 문제를 해결하기 위해 STUN 및 TURN 서버를 사용하는 방법을 학습했습니다.
    - ICE(Interactive Connectivity Establishment): 클라이언트 간의 최적의 경로를 찾는 알고리즘을 이해했습니다.
- 실시간 통신과 대역폭 관리:
    - 네트워크 지연(latency), 패킷 손실(packet loss), 그리고 대역폭 관리의 중요성을 학습하며 WebRTC의 효율성을 높이는 방법을 조사했습니다.

## 3. 기획, 개발

### 1. 아이디어 기획
1. **웹 기반 독서 토론 서비스 북적북적**
- 독서 모임을 만들고 싶었으나 시간/공간적 제약과 모임 진행 경험이 없어서 모임을 만들지 못했던 개인적인 경험이 있습니다. 이 경험에 기반해 Web RTC 기반 독서모임 서비스 아이디어를 제시했습니다.

    1. 제안 배경
        - '23년 통계에 따르면 성인 종합 독서율이 43%로 매우 저조하나, 한강 작가의 노벨상 수상과 텍스트 힙 트렌드로 독서에 대한 관심은 증가하는 추세
        - 관심에 힘입어 독서모임에 대한 수요도 증가함을 확인

    2. 기능 
        - 독서 토론 화상회의 시스템 (그룹 토론)
            - AI의 지원을 통해 그룹 토론에서 더 깊이 있는 논의가 이루어지도록 유도.
            - AI 역할:
                - 주제 제시 및 질문 유도: AI가 주요 주제를 분석하고, 관련 질문을 던져 토론을 이끎.
                - RAG 기반 : 토론 요약, 중요한 발언을 강조하거나, 추가적인 논의가 필요한 부분을 도출 ⇒ 기록 도움
                - 발화자별 발화량을 체크해 모든 참여자가 적극적으로 토론에 참여할 수 있도록 도움
                - 토론 스타일 진단을 통해 더 원활한 토론 스킬을 가질 수 있도록 도움
                - 모임을 마무리할 때에는 다음 모임에서 다룰 책을 추천

        - 추가 기능 및 페이지 아이디어
            - 독서 기록 페이지
                - 기능: 사용자가 책에서 인상 깊었던 구절이나 문장을 기록하고, 해당 구절에 대한 감상이나 생각을 작성할 수 있는 공간.
                - AI 활용: 사용자가 관심 있는 주제에 대해 관련 자료를 추천해주는 기능.
            
            - AI와의 1:1 독서토론 페이지
                - 기능: 사용자가 AI와 개인적으로 독서에 대해 대화하며, 자신만의 속도로 토론을 이어갈 수 있는 공간.
                - AI 활용: 사용자가 질문을 던지거나, AI가 관심있을 만한 주제를 제시하며 상호작용.

            - 독서 토론 피드백 페이지
                - 기능: 사용자가 독서토론 후 피드백을 받을 수 있는 공간.
                - AI 활용: 참가자의 발언 스타일 및 참여도를 분석하고, 개선 사항에 대해 피드백을 제공하여 토론 스킬 향상.

2. **시각 장애인 키오스크 보조 서비스**
    1. 제안 배경
        - 무인 카페, 테이블 오더 등 키오스크 계속 늘어나는 추세.
        - 물리 버튼 없는 키오스크가 많아서 시각장애인들이 불편을 겪고 있음.
            - 2022년 발행한 '장애인 무인정보단말기 접근 이용 모니터링 결과보고서'에 따르면 26개 업종 1천2개의 키오스크를 조사한 결과 648개(64.7％)에 음성·점자표시 등 시각장애인을 위한 편의기능이 없는 것으로 나타났다.
        - 음성 안내 기능이 있는 키오스크가 있어도 보급률이 적고, 가게/기업 비용 부담.
    2. 기능 : 스마트폰 카메라로 키오스크를 인식해서 정보 실시간 전달
        - 키오스크 기계 별 구조 학습
        - 키오스크 구조 → 상세 메뉴 → 결제 순으로 안내
    3. 세부 기능
        1. 실시간 음성 인식, 메뉴 선택 ("카페라떼 주문"이라고 말하면 키오스크에 자동으로 해당 메뉴가 선택)
            - WebRTC를 활용 스마트폰과 키오스크 간 음성 인터페이스 구현
            - 사용자가 말한 내용을 WebRTC를 통해 음성 스트리밍 → 음성 인식 후 텍스트로 변환 → 키오스크 입력으로 전달.
        2. WebRTC를 통한 데이터 공유 및 기록
            - 사용자가 키오스크 이용 중 입력한 데이터와 사용 패턴을 WebRTC를 통해 클라우드에 실시간 저장
            - 이후 사용 이력을 바탕으로 개인화된 메뉴 추천 제공.
            - 사용자는 반복 작업을 줄이고, 더 빠르게 결제 가능
